1. OBJETIVO
================================================================================

Gerar dataset CFD 3D transiente do escoamento ao redor de cilindro aquecido
para uso como ground-truth no treinamento de Physics-Informed Neural Network
(PINN) em TensorFlow.

Premissa metodológica: validar a metodologia PINN mesmo com física laminar
não-realista (Re alto efetivamente Euler), priorizando reprodutibilidade
arquitetural sobre fidelidade fenomenológica.

5. EXPORT DE DADOS - DESCOBERTA CRÍTICA
================================================================================

5.1 PROBLEMA IDENTIFICADO
Tentativas iniciais de export geravam ~55k boundary nodes em vez dos cell
centers volumétricos esperados. Diagnóstico via análise das velocidades:
u=0 e w=0 em 100% dos pontos (sinal de inicialização sem solver rodado) +
todos os pontos a <0.5m de superfícies.

5.2 CAUSA RAIZ
Ao selecionar Surfaces no diálogo Export → Solution Data, o Fluent aplica
um filtro que pega apenas cell centers cujas células TOCAM essas surfaces
(boundary layer). O restante do volume é descartado.

5.3 CONFIGURAÇÃO CORRETA - EXPORT MANUAL
File → Export → Solution Data:
- File Type   : ASCII
- Location    : Cell Center  (CRÍTICO - sem este check exporta nodes)
- Cell Zones  : domain selecionado
- Surfaces    : VAZIO (Deselect All)
- Quantities  : Pressure, X Velocity, Y Velocity, Z Velocity
- Delimiter   : Comma
- File Name   : /caminho/absoluto/arquivo.csv

Resultado: ~213.000 linhas (igual ao cell count da malha).

5.4 CONFIGURAÇÃO PARA EXPORT AUTOMÁTICO DOS 1000 SNAPSHOTS
Solution → Calculation Activities → Execute Commands:

  Active     : ✓
  When       : Time Step
  Every      : 1
  Command    : /file/export/ascii "CSVs/timestep_%t.csv" no domain () no
               yes pressure x-velocity y-velocity z-velocity quit

Decomposição do comando TUI:
- /file/export/ascii   → comando
- "CSVs/timestep_%t.csv" → nome do arquivo (%t = timestep automático)
- no                   → não exportar pontos das surfaces (só interior)
- domain ()            → cell zone fluida + lista vazia (fim)
- no                   → não escrever node values (forçar cell center)
- yes                  → escrever coordenadas
- pressure x-velocity y-velocity z-velocity → quantidades
- quit                 → fim da lista
- (sem confirmação no final do comando para evitar prompt)

NOTA: Fluent ignora extensão .csv em export ASCII por padrão. Necessário
forçar a extensão no nome OU renomear em batch após simulação.

11. LIMITAÇÕES E TRABALHOS FUTUROS
================================================================================

- BOI_Triangle subdimensionado: 9.6% (alvo era 15-20%). Esteira distal
  recebe menos resolução que o ideal. Para Re menor (turbulência visível),
  reduzir BOI_T para 3.5-4.0m

- Adj. BCs abaixo do alvo: 11.2% (alvo 20%). Particularmente laterais
  com sizing 7m têm pouca cobertura. Para validação de BCs sem HC,
  refinar laterais para 5-6m

- Sem inflation layers: para problemas turbulentos reais, falta malha
  estruturada nas primeiras camadas das paredes

- Cylinder_top com sizing igual ao wall: para estudos detalhados,
  considerar sizing separado (~2-3m) para o disco superior

- Sem refinamento adaptativo: ativar Mesh Adaption no Fluent baseado em
  gradiente de velocidade após iterações iniciais (próxima versão)

- Estudo de independência de malha não formalizado: necessário gerar
  três malhas em fator √2 de refinamento e comparar grandezas integrais
  (Cd, Cl, St) antes da defesa final

- Domínio downstream curto: y=700m vs cilindro em y=200m dá ~9D de
  esteira. Para Re alto com von Kármán bem desenvolvido, recomenda-se
  15-20D downstream

- Skewness máx 0.48 no surface mesh: aceitável mas há células de transição
  BCs→BOIs distorcidas. Para convergência turbulenta, aumentar GR=1.3 ou
  adicionar inflation

13.1 PARÁGRAFO METODOLÓGICO PARA O CAPÍTULO

"A malha computacional foi gerada com Fluent Meshing utilizando elementos
poliédricos via Watertight Workflow. O dimensionamento adotou face sizing
local nas paredes (cylinder: 1m; inlet/outlet: 4m; laterais: 7m; top/ground:
8m), Bodies of Influence para refinamento volumétrico nas regiões de interesse
(BOI Cilindro: 4.5m; BOI Triangle: 4.8m) e Max Cell Length 25m no freestream
global, com taxa de crescimento 1.2 e Min/Max Surface Size 0.5/15m.

A malha resultante contém 213.146 células poliédricas com qualidade ortogonal
mínima de 0.37, atendendo aos critérios do solver Fluent (>0.20) com folga.
A distribuição volumétrica dos centroides das células — utilizados como pontos
de dados para o PINN — é de 19.0% adjacentes ao cilindro, 20.9% no BOI Cilindro,
9.6% no BOI Triangle, 11.2% adjacentes às BCs externas e 39.3% no freestream,
configurando refinamento progressivo das regiões com gradientes físicos
significativos.

Os dados utilizados como ground-truth para treinamento do PINN são os valores
resolvidos diretamente pelo solver Fluent (Finite Volume Method), correspondendo
aos centroides das células poliédricas. Esta escolha preserva a fidelidade da
solução numérica, evitando o uso de valores interpolados nos nós da malha, que
introduzem aproximações de segunda ordem decorrentes do processo de reconstrução
pós-solver."
